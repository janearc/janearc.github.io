lib/js/sendak.js

the Sendak back-end contains all the calls necessary to translate between the various objects/interfaces used by 18F's users: github and AWS, with the Riak/rm backend to store the metadata/intersection between those systems, as well as the "policy" or "identity" data. This functionality is exported in the Sendak object. Hierarchically, there is a 'users' object, underneath which are 'sendak', 'aws', and 'github' objects. Their interfaces share the same argument names and taxonomy; Sendak.users.aws.get takes the same arguments as Sendak.users.github.get, and provides the same objects (albeit with some additional/fewer fields).

Example

Sendak.users.aws.get( {
     'user-name': 'JaneArc',
     'arn': true,
   } ).then( function (awsuser) {
     Sendak.users.sendak.get( {
       'arn': awsuser.arn,
       'given-name': true,
       'surname': true,
     } ).then( function (sendakuser) {
       Sendak.stdout( ''.concat(
         'Hello ',
         sendakuser['given-name'],
         ' it looks like your arn might be ',
         awsuser['arn'],
         'and your userid is ',
         awsuser['user-id']
       ) );
     } );
   } );
